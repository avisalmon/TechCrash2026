How They Connect

The ESP32 and DE10-Lite communicate over UART (serial) via the Arduino header. Two wires + GND, bidirectional, 9600 baud. Use male-to-male jumper wires.

ESP32 → FPGA

ESP32 GPIO 16 (UART2 TX) connects to FPGA GPIO[0] (Arduino IO0). The ESP32 sends data to the FPGA.

FPGA → ESP32

FPGA GPIO[1] (Arduino IO1) connects to ESP32 GPIO 17 (UART2 RX). The FPGA sends data back to the ESP32.

Common Ground

Connect ESP32 GND to Arduino header GND. Both boards must share a common ground for UART to work.

Ground via VGA Port

No spare jumper wire? Grab GND from the DE10-Lite VGA connector (DE-15). Pins 5, 6, 7, 8, and 10 are all ground — all connected together. Pins 6, 7, 8 (middle row, left side) are easiest to reach. Use a female-to-female jumper or poke a bare wire into any of those sockets.

Get Started

Everything you need to go from unboxing to building. Complete the installation before May 13 and arrive with a working laptop.

Do This Before May 13

Do these four things first. If these are done, the rest of the page becomes much easier.

1. Install Quartus 17.1

Install Intel Quartus Prime Lite 17.1 with ModelSim and MAX 10 device support, plus the USB-Blaster driver.

2. Install VS Code and PlatformIO

Install Visual Studio Code and add PlatformIO IDE so the software environment is ready before kit pickup. The actual ESP32 build and flash steps happen after you have the hardware in hand.

3. Prepare GitHub and Copilot

Create a GitHub account, apply to GitHub Education with your Technion email as early as possible, and sign in to Copilot in VS Code.

4. Bring a Ready Laptop

Arrive with a laptop that already opens the tools correctly, has working USB ports, and is ready to connect to the hardware kits.

Ready to Arrive?

Use this checklist before you leave home. If you cannot check these off, fix that first.

  • Quartus Prime Lite 17.1 opens on your laptop
  • VS Code opens normally and the PlatformIO extension is installed
  • You can sign in to your GitHub account from VS Code
  • GitHub Copilot is installed and signed in
  • Your GitHub Education request has already been submitted, or approved
  • You have a laptop charger and at least one working USB cable
  • You are available for both prep meetings: May 13 and May 26

Your Setup Checklist

Complete these steps in order. Each one builds on the previous.

1

Pick Up Your Hardware Kit

Collect your DE10-Lite FPGA board and ESP32 controller kit at the May 13 session. Check that you have USB cables, jumper wires, and a breadboard. If a registered team does not claim a kit by 13:00, it may be reassigned to the waiting list.

2

Install Quartus Prime Lite 17.1 (FPGA)

Download and install Intel Quartus Prime Lite 17.1 with ModelSim and MAX 10 device support. Install the USB-Blaster driver so you can program the DE10-Lite.

  • Download: Quartus Prime Lite 17.1 for Windows
  • Select: Quartus + ModelSim + MAX 10 device support
  • Install to: C:\intelFPGA_lite\17.1\
  • USB-Blaster driver: C:\intelFPGA_lite\17.1\quartus\drivers\usb-blaster

See the full guide in the de10lite-board-and-build skill

3

Install VS Code + PlatformIO (ESP32)

Install Visual Studio Code, then add the PlatformIO extension. PlatformIO prepares the ESP32 software toolchain in advance, but the actual board build, flash, and hardware verification steps happen only after you receive the ESP32 kit.

  • Download VS Code: code.visualstudio.com
  • Install extension: search "PlatformIO IDE" in Extensions
  • Install USB driver: CP2102 or CH340 (check Device Manager)

See the full guide in the esp32-firmware skill

4

Set Up GitHub Copilot

GitHub Copilot is your AI engineering teammate. Install the extension in VS Code and sign in with your GitHub account. Use it for SystemVerilog, C/C++, and everything in between.

  • Install: search "GitHub Copilot" in VS Code Extensions
  • Sign in with your GitHub account
  • Apply to GitHub Education with your Technion email as early as possible. Approval may take a few days.
  • The organizing team will also provide hackathon access before the event.
5

Test Your FPGA — LED Smoke Test

Create a minimal Quartus project that maps the 10 slide switches to the 10 LEDs. Compile, program the board, and flip switches. LEDs light up = your FPGA toolchain works.

Working example: Alive Test demo in the repo (demos/alive_test/fpga/) includes a complete Quartus project you can compile and program.

6

Test Your ESP32 — OLED + Button + Buzzer

After kit pickup, create a PlatformIO project that displays text on the OLED, detects button presses, and beeps the buzzer. Build, flash, and verify all three peripherals work only once the ESP32 kit is in your hands.

Working example: the Alive Test demo ESP32 code (demos/alive_test/esp32/) shows how to set up PlatformIO and communicate with the FPGA.

7

Connect FPGA ↔ ESP32

Wire the two boards together using jumper cables (UART). Send data from one to the other. This is the core skill for every challenge.

Tip: Start simple: UART TX from ESP32 GPIO16 → FPGA Arduino IO0. One wire + ground.

Working demo: Alive Test — bidirectional UART with full source code, wiring diagram, and block diagram. Clone the repo and open demos/alive_test/ to see both the FPGA and ESP32 projects side by side.

8

Try a Practice Challenge

Head to the Practice Demos tab and study the working demos. Then try to build your own variation or extend them.

The Internet Clock demo (demos/internet_clock/) has a working FPGA + ESP32 project you can study for inspiration. Use GitHub Copilot to help you build the rest.

AI Tools

In this hackathon, AI isn't a shortcut — it's a core tool. You're expected to use it. The best teams will be the ones who learn to work with AI the fastest. Here's your toolkit.

Why AI in a Hardware Hackathon?

Speed

Generate boilerplate SystemVerilog modules, pin assignments, and C firmware in seconds instead of hours. More time building, less time typing.

Exploration

Don't know how UART works? Ask Copilot to explain it and generate an example. AI lets you explore unfamiliar territory fast.

Debugging

Paste a compile error or a timing waveform description — Copilot can help you reason through it and suggest fixes.

Learning

No tutorials, no instructors. AI is your on-demand teacher. Ask it anything — from Quartus project setup to SPI protocol details.

Visual Studio Code

Your main development environment for both FPGA and ESP32 work.

Multi-Language Editor

SystemVerilog for FPGA, C/C++ for ESP32, Python for scripts — all in one editor with syntax highlighting and IntelliSense.

PlatformIO Extension

Build, flash, and monitor your ESP32 directly from VS Code. No command line needed.

Integrated Terminal

Run Quartus commands, serial monitors, and git — all without leaving the editor.

Git Built-In

Version control your project from day one. Commit often, branch freely, never lose work.

Install: code.visualstudio.com — free, all platforms.

GitHub Copilot

Your AI pair programmer, built into VS Code.

Copilot Chat

Open the chat panel (Ctrl+Shift+I) and ask anything: "How do I create a UART receiver in SystemVerilog?", "Explain this error", "Generate a PlatformIO project for ESP32 with OLED".

Inline Completions

Start typing code and Copilot suggests the rest. Write a comment describing what you want, press Tab, and watch it appear. Works with SystemVerilog, C, C++, and more.

Copilot Edits

Select code, press Ctrl+I, and tell Copilot what to change: "Add a reset signal", "Convert this to a state machine", "Fix the timing". It edits your code in-place.

Agent Mode

Give Copilot a high-level goal and let it work autonomously — it can read files, run terminal commands, search your codebase, and iterate until the task is done.

Pro Tips for Effective Use

  • Be specific: "Generate a SystemVerilog module that receives UART at 115200 baud on GPIO pin 12 and outputs 8-bit data" → better results than "make a UART".
  • Provide context: Open relevant files in your editor. Copilot reads your open tabs to understand your project.
  • Iterate: First answer not perfect? Say "make it synchronous" or "add error handling". Copilot refines.
  • Attach files: In Chat, use @workspace to let Copilot search your entire project, or drag files into the chat.
  • Ask it to explain: Don't just generate — ask "explain this code line by line" to actually learn.
Install: Search "GitHub Copilot" in VS Code Extensions → Install → Sign in with GitHub.
